This is an example of complicated fancy indexing. A lot of the magic happens in line 15, but there are some other interesting nuggets to be found, like vectorizing the stringification of the result. Vectorization here doesn't necessarily make the code faster, but it does make it cleaner (lines 16 and 17). Fancy indexing will definitely make your code faster, cleaner and less error prone.

Here, we first take the median of an array and then apply the fizzbuzz algorithm to it.

import numpy as np # 1 as_strided = np.lib.stride_tricks.as_strided # 2 x=7 # 3 input_arr = np.arange(x*x).reshape(x,x) # 4 # x-2 because there is no padding. Doesn't make sense for median # 5 n = x-2 # 6 strided = as_strided( # 7 input_arr, # 8 shape=(n, n, 3, 3), # 9 strides=(input_arr.strides[s] # 10 for s in [0, 1, 0, 1])) # 11 m = np.median(strided,axis=(2,3)) # 12 v = np.array([ # 13 ' ', ' Fizz', ' Buzz', 'FizzBuzz' ]) mod = np.array([1,3,5,15]) # 14 answer = v[3-np.argmin((m[:,:,None]%mod)[...,::-1] > 0,axis=2)] # 15 str_fn = lambda i: str.rjust(str(int(i)),8) # 16 str_vec_fn = np.vectorize(str_fn) # 17 empty = np.where(answer == ' ') # 18 answer[empty] = str_vec_fn(m[empty]) # 19 print (input_arr.astype(float)) # 20 print (m) # 21 print (answer) # 22

Pay special attention to line 15.
Here is the explanation of each element of line 15.

1.) v[3-np.argmin((m[:,:,None]%mod)[...,::-1] > 0,axis=2)] For reference. 2.) _______________m[:,:,None]____________________________ Here, we extend m, our median matrix to accomodate a 3rd dimension, making the array (5,5,1). This dimension will be length 1, but will allow us to apply the (1,4) modulus operator on m for each (5,5) element of m, eventually resulting in a (5,5,4) array. 3.) ______________(m[:,:,None]%mod)_______________________ Now we actually perform a modulus operation on the expanded median matrix m. There are two shapes: (5,5,1) and (1,4). The modulus operator will apply the 4 values of 'mod', ([1,3,5,15]), along the 5x5 elements of m and store them in the new dimension we created. For this to work, the size of the last dimension of the first matrix and the size of the first dimension of the second matrix must be the same. (_,_,1) and (1,_) in our case. The resulting matrix will be (5,5,4). 4.) ______________(m[:,:,None]%mod)[...,::-1]_____________ We are going to reverse that matrix along the 3rd axis. The reason is that we need to find the smallest value, but at the highest index, since in [1,3,5,15], 3 has a higher precedence over 1, 5 has a higher precedence over 3, and 15 has a higher precedence than 5, according to the rules of fizzbuzz. If we didn't do this, our argmin would have the precedence reversed. Note, we could have eliminated this step if we had defined `mod` in the reverse order [15,5,3,1]. 5.) ______________(m[:,:,None]%mod)[...,::-1] > 0_________ Now, we are just converting all values to true/false. We are going to use argmin and find the location of values that are not zero, i.e. those values that are not evenly divisible by the modulus value. 5.) ____np.argmin((m[:,:,None]%mod)[...,::-1] > 0,axis=2)_ Almost there! We now find the indexes where the values are not false. Only the first index that matches the min criteria is returned along axis 2. We'll do that along the 3rd dimension, which is our modulus results dimension. The values will be between 0 and 3, corresponding to the REVERSED order of the modulus operators. (Because we flipped the order previously). 6.) __3-np.argmin((m[:,:,None]%mod)[...,::-1] > 0,axis=2)_ Now we subtract those REVERSED indexes from 3, to unreverse them. We now have an array that is in the same shape as our answer (5x5) and ranges between 0 and 3, which will serve as indices to values in the word array. 7). v[3-np.argmin((m[:,:,None]%mod)[...,::-1] > 0,axis=2)] Here, we are going to pull elements from the v array of words using fancy indexing. The index array that goes here is in in the shape of our final result (5x5), so for every element the answer will be populated with words corresponding to the indexes (0 to 3) pulled from the `v` array. Therefore, our answer will be 5x5, even though the word array is only (4,)!
